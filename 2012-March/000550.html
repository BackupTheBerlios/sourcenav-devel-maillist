>> For this process to parse the project automatically I've been set my
>> sourcenavigator path to the windows system path variable and then tried to
>> do "sourcenavigator.exe --batchmode", but I got only the error message
>> window "Failed to start Source Navigator 4".

Ah, yes. This doesn't work as the setting up of the dll search path
and general SN environment (equivalent of $LD_LIBRARY_PATH and
$snbasedir under *nix) does not work if snavigator.exe is invoked via
%PATH% under windows. While in theory, DLLs should be searched also in
%PATH% -- in SN's practice this does not really work.
You can have a look into SRC/snavigator/snavigator/win/snavigator.c
(IIRC), and also into the tcl startup files (name currently escapes
me), if you're interested.

http://svn.berlios.de/viewvc/sourcenav/trunk/snavigator/snavigator/win/snavigator.c?annotate=725

As for your endeavours: you could also directly invoke a parser
(.../libexec/snavigator/cbrowser for example) , if you'd like to.
You'd get funny output on stdout, detailed in SN's progref manual --
global variables should be denoted by "gv" IIRC.
But again, on win32, you will get startup problems due to missing
environment to find according DLLs.
