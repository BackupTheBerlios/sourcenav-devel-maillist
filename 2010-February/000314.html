>  Being a c, c++ embedded programmer I haven't learned the tcl syntax or made the time to
>  debug it myself. Is there a debug mode I can build to produce a lot of spewage?

You can try the configure options

--enable-symbols=mem

to enable tcl memory allocator debugging (to catch potential
overwrites). HOWEVER db4 doesn't use that allocator, so you'd only
catch overwrites on non-db4 subsystem memory.
You also can try to start SN with cmdline option "-debug 99", this
will spew a lot of stuff to both a logfile (zznavig.log) and to
stdout.
