<background>
One of the changes made between the old and new versions of the
autoconf tools used by SN is the flip of the default behavour re
caching

The old version had "always use cache" (or some such switch) defaulted
to NO/FALSE
The new version defaults to YES/TRUE

This uncovered a couple of bugs in the input files to autoconf etc.  I
sorted out what barfed on the Mac OS X and assumed that was all.
Perhaps the mingw platform is subtly different and is uncovering more
problems in this area.
</background>
