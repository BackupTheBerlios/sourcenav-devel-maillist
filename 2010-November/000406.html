Hi Markus,

>>> It is not possible to change the color and fonts for certain windows.
>>[...]
>>>> Open Symbols window -> File Project Preferences.... -> Colors&Fonts ->
>>> Select "Grep" in the smaller window to the left-> Select "Listing"
>>> under Grep -> To the right, try to change "Foreground" and
>>> "Background" color, it doesn't work.

This issue seems to be per design - the options for fg and bg are
explicitly disabled in the code; I think this is due to the fact that
the listing itself is a listing (and its colors are to be defined
elsewhere, namely for all standard listings), and furthermore the
grep-listing cannot be locally changed because it's a standard listing
(duh!).

You can look for yourself, the code is in
snavigator/gui/preferences.tcl, line 1596.
The variables for fg and bg are set to  "" , instead of something like
grep-listing-fg and grep-listing-bg. However adding these means adding
them in all settings save/load functions and in the widget
instantiations...

For the moment I'd like to not follow this issue and focus on other,
more serious errors first.
Nevertheless if you can contribute a patch of course I will
immediately merge it ;)
