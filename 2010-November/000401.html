Hi,

The grep function/window appears not to be able to jump to a match for all strings. For some text/strings it jumps correct to the match in the source code when double clicking on the search matc, for some it doesn't jump at all.

Here is an example how to reproduce the fault:

1) Create a text file (call it anything, for example asdf.txt) containing the following text:

asdf MUX
counter_id = asiRead16_lookup(MUX_DEMUX_ARRAY_BASE:5, $session_id:16, VIRTUAL_HANDLE:1, 0:2)
asiWrite32_lookup(MUX_DEMUX_ARRAY_BASE:5, DEMUX_SESSION_ID_11:16, VIRTUAL_HANDLE:1, 0:2, MUX_DEMUX_VIRT_11:32)
vbnm MUX

2) Start source navigator and create a project and add the file asdf.txt to the project.

3) Open the grep window and grep for MUX.

4) Now double click on the matches in the "grep window", for some of the matches the editor jumps automatically to the match, but for some of the matches it doesn't jump to the match.
