Hello Mr. Kiefer.

The basic problem is that the current SN Java parser
is based on the 1.0 Java specs, and what we have are the
program generated code (which is unmaintainable). So,
what is needed is to create the parsing rules using
opensource tools (flex with perhaps bison) with the
more recent Java specs.

What's happening is that newer Java constructs, like Enum,
are not being handled properly by the program jbrowser.
The code is causing a segfault as it tries to bring back
its state machine back to a sane state after encountering
something new.

Freek can start you up with an SVN branch to play with.
As a point of reference, I can help you along the way
with how the PHP browser is constructed using the SN
library and opensource tools.

Hope you have time to help. Good luck.

--Maurice
