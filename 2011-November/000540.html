Bruce Simpson wrote:
> Hi,
>
> I would like to know why Source Navigator is tied to specific
> versions of Tcl/Tk?
>
> These packages are commonly pre-installed on most modern
> distributions or vendor systems. It would help with trying to build
> it on other systems e.g. MacOS X 10.7.

The reasons predate the current work but typically this is done to
ensure that the software continues to work when someone decides to
install a newer version that breaks API compatibility.  Recently
however, shared libraries of specific versions contain an API version
number, using that specific version number should be doable but may be
too time expensive for the current maintainers to manage so the work is
toward improvements in other places.  Frank will correct me if I'm wrong
here.
