Hi,

As usual as was completely wrong about the cause of the crash in Windows, it
so happens that the YY_INPUT macro in tcl_high.c was preempted by another
definition in highlight.h, following that I got to a place where SN loads the
tk dll, for whatever the reason.

The code to decide the name of the dll was:

sprintf(dllname, "rhtk%d%d%s.dll", TK_MAJOR_VERSION, TK_MINOR_VERSION, TK_DBGX);

Since I hadn't bothered with the red hat prefix, the program failed to load the dll
and crashed.

That means that port now works in Windows, not flawlessly, but it works, kind of at
least.

If you want to take a look, you can find the source tarball:

http://sourceforge.net/projects/getleftdown/files/SourceNavigator-Tryouts/SourceNavigator-NG-8.4-3.tar.gz/download

And the windows installer:

http://sourceforge.net/projects/getleftdown/files/SourceNavigator-Tryouts/sourcenav-ng-setup-v4.4.91.exe/download

And now that I have gotten that crash off my chest, I am going to look into creating a 
git repository for the port.
