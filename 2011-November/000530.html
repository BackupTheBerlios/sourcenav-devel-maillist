Hi Andres, gentlemen -

please wait until you fork/clone our repository - the code on github
is only experimental, where I tried the workflow features.
It is in no way code that SN WILL contain. Other than that such a
clone is harmless and does not mess with us.

I will prepare a new git repository, and then you can fork and prepare
your adjustments in a seperate branch. It's always nice to do it
kernel-style: "for-linus-performance-work" or
"feature-worlddomination" or somesuch.
In our (previous) SVN days I did a merge (squash) commit, whereas with
GIT I usually like to do a fast-forward merge, so you will see the
individual commits; therefore it is extremely important to have small,
structured changes to the code, i.e. commit #1 is introduce new
function and commit #2 is use new function and commit #3 is cleanup
redundant code now.
Only if I don't like something or I want to squash smaller changes
into one big chance I will do a squash merge.
