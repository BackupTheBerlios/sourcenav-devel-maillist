> I have tried without success to build sourcenavigatorNG3 on Mac OS 10.5.6.
>
> The make process mostly uses gcc for compiling, but in some cases cc is used
> for linking programs, resulting in an error message because the "-soname"
>  option isn't supported.

OSX is still a bit of a rough platform, but Mark was on it a while
back and we had merged some patches for it, too.

We had manually fixed that a while back, but rebuilding all the auto*
stuff seems to have reverted it to the old behaviour (cc is tried
first instead of gcc in db4/ directory).

If you have a bash on OSX, you can try with

#> CC=gcc ./configure --prefix=/some/where <other_flags>

That should do the trick for your build. If not, try attached patch.
I will try to fix this in a more permanent way, too.
