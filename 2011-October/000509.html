Here is wiki's write up ...In computer programming, a *widget* (or *control*)
is an element of a graphical user
interface<http://en.m.wikipedia.org/wiki/Graphical_user_interface>(GUI)
that displays an information arrangement changeable by the user, such
as a window <http://en.m.wikipedia.org/wiki/Window_%28computing%29> or a text
box <http://en.m.wikipedia.org/wiki/Text_box>.

In GUI dev world, there are hundreds of widgets representing menus, buttons,
frames, text areas, radio buttons, etc. (Think HTML elements.) Which are
rendered by some widget rendering layer which provides a different look and
feel. Sort of like an HTML tag (or element) that is rendered one way on
Firefox and different way on IE, etc

So just like there is a family feud on browsers, there is a family feud on
XYZ Widgets vs ABC Widgets. You then take this feud and sprinkle a bit of
Geometry Manager dispute and you get a nice wrestling match. Just to add a
bit of taste to it, you add the notion of Perl vs Tcl, vs C++, vs Java and
you have a nice backyard rubmble.

> ttk widgets are Themed Tk widgets, they were added at the Tcl/Tk core
> for version 8.5 and are also available as an extension for 8.4.
>
> Ttk uses native widgets in the different platforms, either Windows, with
> vista, xp and winnative themes, macs and Linux.
>
> You can get an idea of what it means to the end user with some screenshots
> from a program of mine:
>
> http://personal.telefonica.terra.es/web/getleft/snaps.html
>
> Although those snaps have three advantages:
>
>        - Tcl/Tk 8.5 which means antiased fronts.
>
>        - It uses the tkpng extension so the icons are pngs not gifs.
>
>        - It uses the TileQt theme which makes ttk use Qt widgets,
>          unfortunately, it is no longer being maintained.
>
> Porting SN to use them also means you can get rid of Tix and,
