Yes, it's unusual, yet used here and there (and I guess it's standard compliant too).
Anyway, SN behaves the same for more usual/standard code like this:

#include <string>

class string_ex : std::string
{
};

Declaration of string is found but class tree for string_ex is broken (std::string is not found).
To reproduce this on Linux, make a new project from /usr/include/c++/4.2.1 (or whatever gcc version there is on your machine), add a file with the string_ex code from above, select "string_ex" in the editor and navigate to its class hierarchy.
