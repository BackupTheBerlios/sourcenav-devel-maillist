Hi Alex,

well, officially C# is not supported. Currently we also don't have the
manpower to write just another language parser (and quite frankly, what the
hell do *I* care about a non-free, non-standard, windows-only programming
language? I don't want to start a flamewar here, but sorry - as Linus I've
also got strong opinions).

But you could try to feed the code into the C/C++ parser anyways.
This will of course mean that it will not recognize all of C#'s features.
Find the file "sn_prop.cfg" at
<SN_INSTALLROOT>/share/snavigator/etc/sn_prop.cfg
then go to the following section:

# C++
if {$tcl_platform(platform) == "windows"} {
        #windows file dialog box doesn't understand with '[]' grouped
extensions
        set cpp_ext {*.h *.H *.c *.C *.cxx *.hxx *.cpp *.c++ *.hpp *.cc *.hh
*.CXX *.HXX *.CPP *.HPP *.CC *.HH}
} else {
        set cpp_ext {*.[hHcC] *.[ch]xx *.[ch]pp *.c++ *.cc *.hh *.[CH]XX
*.[CH]PP *.CC *.HH}
}
sn_add_parser c++ \
        -suffix $cpp_ext \
        -brow_cmd $sn_path(parserdir)/cbrowser \
        -macros "-m"

and then add the .cs file extension into the list (or whatever extension
you're using).
However, I have no clue as to what will happen, so don't blame me ;-)

Please report back and cheers,
