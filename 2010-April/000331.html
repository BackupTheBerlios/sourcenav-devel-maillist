Well, no - you don't build it like that. You build it by using
MinGW/msys and doing a good old "configure --prefix=/some/thing ; make
; make install". Just look into INSTALL, it's all there.

As for tcl/tk I'd strongly suggest to get familiar with it, because
that way you'll get the quickest results and tcl is SN's main
language.
Please note that while SN builds and runs on win32 it is by no chance
our best supported platform. It's much faster and easier to work with
(development-wise) on Linux.

what you'd expect from SN re: parsing/xref/whatver, and also a
download link to more example code (if any).
