> Hi Wolfgang,
> 
> yes, you can query the db. No, it's not really that easy :)
> At first I suggest you'd get the SN source code because the win32 and
> linux builds don't contain any of the development files (headers,
> libraries, whatnot).
> 
> And then you can start here:
> http://sourcenav.sourceforge.net/online-docs/progref/dbaseAPI.html#pgfId-996751
> This describes the DB API (mostly via tcl).
> 
> You should also have a look at $SNSOURCE/snavigator/sdk/database and
> the tcl files in there; they will show you how to query and especially
> combine db information, esp. call-freq.tcl, unused.tcl and maybe
> diamonds.tcl (they need to be started with hyper, so you will have to
> fiddle around with a startup script similar to snavigator).
> Other than that the "sdk" isn't really in a great shape.
> 
> If you want to go lowlevel you should look at dbutils.c and dbimp.c
> and also should manually run a parser to see what the parsers really
> spew out for you to use.
> It's like: parser(file.c) --pipe--> dbimp --> SNDB4/<files>
> 
> What you absolutely don't want to do is reimport the db stuff, trust
> me - SN has quite a nasty db schema.
> We'll also love to hear what you've achieved and how - just report
> back if you feel like it. Or if you can contribute something along the
> way :)
> 
> Cheers
