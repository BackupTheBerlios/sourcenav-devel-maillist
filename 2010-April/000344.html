> Thank u Freek, do u know a method for debbugging the entire sourcenavigator
> code? So I'll understand where the DB layer (and  other stuffs) are.

The DB layer is split in 2 parts:
- the C code is in snavigator/db. A lot of shared functions are in
dbutils.c, whereas the actual DB import process is in dbimp.c (reading
via stdin from another forked parser process). The rest you can ignore
(dbcp et al).
- the wrapper tcl code is in hyper/dbsym.c and others - it makes
certain C functions available to tcl.

Good luck, you'll need it ;-)
