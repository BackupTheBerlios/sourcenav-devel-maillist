Hello,

Does anybody know whether some of the arguments of the API functions (sn_insert_xref, sn_insert_symbol & the put_* used by these) are actually namespaces?  Any comments or insights are appreciated. 

I am looking into adding namespace support to SN and I have stumbled over such code snippets:

In ibroswer.c, I can see the following mention of namespaces:

     sn_insert_xref( SN_REF_TO_CONSTANT	/* referenced symbol type	*/
	    , SN_FUNC_DEF		/* scope type			*/
	    , SN_REF_SCOPE_GLOBAL	/* scope level			*/
	    , NULL			/* class name			*/
	    , last_label.name		/* calling function name	*/
	    , NULL			/* argument types		*/
	    , NULL			/* namespace of symbol		*/
	    , p				/* referenced symbol		*/
	    , NULL			/* referenced argument types	*/
	    , sn_current_file()		/* file name			*/
	    , sn_line()			/* line number			*/
	    , SN_REF_READ		/* access type			*/
	    );


But in snptools.c, I see that the namespace is actually the referring class: 

int sn_insert_xref(int type, int scope_type, int scope_level,
                    char *classname, char *funcname, char *argtypes, 
                    char *refclass, char *refsymbol, char *ref_arg_types,
                    char *filename, int lineno, int acc)


Is ever refclass used as a namespace holder?
