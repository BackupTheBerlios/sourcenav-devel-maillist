Did you build GCC yourself or get it from a third party?  You might 
want to try using the native compiler (all unix platforms have a 
compiler for OS patching).

> configure:3594: gcc -qlanglvl=ansi -c -g -D_REENTRANT conftest.c >&5

> /usr/local/hplx/include/hp_include/stdio.h:593: parse error before "__size_t"

The stdio.h issue is usually an indication that some expected macro 
isn't defined.  The configure command is probably trying out different 
options to see if it is valid but I don't know that.  If you started 
the build process without CC set then you need to start over with CC 
set for the entire build.
