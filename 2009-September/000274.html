Hi Chris,

first - sorry I don't have any Vista to cross-check this.
Second - yes, SN will scan the source directory and NOT show the gui
on huge source trees. This has partly to do with the slowness of NTFS
regarding meta-operations (getdents(), stat() etc) and partly with a
funny tk bug where the GUI is not shown despite it's commanded to do
so.
If you're capable of tcl/tk you might look into the snavigator/gui
directory, file projmng.tcl.
You'll see funny sleeps around the place and some references to
windows and buggy behaviour.

You could work around this by selecting the source tree in the new
project startup, but instead clicking on "Project editor".
It will ask you if it should pre-scan for all the files (say yes) and
then in the next dialogue (the project editor) check that the source
tree is imported and click OK to start the indexing and
cross-referencing.
