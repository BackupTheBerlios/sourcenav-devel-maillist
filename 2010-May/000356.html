Hi Fabio,

maybe I can help a little: the parsers are stand-alone executables. They are executed by a kind of "system"-call done by SN. The executable dbimp takes the output from a parser and fills the project database, which is then accessed by SN again. Which browser is used for which programming language is configured by file extensions. The C++-browser is a bit different in that the work is split among two executables: cbrowser2 generates the xref-info based on an xref-file generated by cbrowser -x xref-file. (And there are some more undocumented command line options of cbrowser.)

Hope that helps,

Wolfgang
