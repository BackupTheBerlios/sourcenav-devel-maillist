> 3. why even a printf seems to has no effect (probably because the executable

If I remember correctly (in all the parsers) the file descriptor for
stdout (which printf will operate on) has either been closed or
remapped to the pipe "dbimp" is reading from.
So you wont see any output. I also seem to see this problem on win32
with the "normal" hyper process, but I couldn't find out what the
problem was.

Quick workaround: open a file "/tmp/mydebug" and write all your
debugging output there with fprintf().
