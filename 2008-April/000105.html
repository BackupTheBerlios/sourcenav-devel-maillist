>  3) The source package still contains its own version of tcl/tk. I think
>  it could become very attractive for distributions to once again include
>  source-navigator, if this was resolved. I don't think distros like
>  packages that forces the user to install outdated versions of other
>  packages. I would love get SN via YUM or apt-get in the future!

This is not a big showstopper for inclusion into repositories: SN is
strictly linked statically against tcl/tk so it doesn't disturb other
packages. The packager can even remove those static libs (they're
provided to have a full SDK included) for a "run-only" package.

However, yes - we WERE thinking about removing the "own" version of
tcl/tk or upgrading the supplied version to 8.4 but then SN is hugely
version depended on the version of tcl/tk by using itcl and also some
custom widgets libraries.
Which all blows up big time when doing a switch to 8.4 (or even using
the underlying OS's version).
So we came to the conclusion to NOT change this at the moment.
