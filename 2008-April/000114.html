Hi John.

> I suspect that the file locking that is used was a compromise for something that was portable across various unix and windows platforms.
I am running SN-NG from SVN on fedora-8.

At least on unix, i think the system lock on the file should be enough.
I don't see any obvious point of holding a custom lock beside that.
If there is, is should probably be disabled on unix targets, as
these targets have a solid locking mechanism built-in.

On win32/win64 (and others) a custom lock might be a good idea,
as locking can get quite funky on those systems (poor FS design).
However, i think it is resolved on newer systems like XP and Vista.

Anyway, when using SN on win32 i have usually used it in Cygwin,
which in this case should pass for a unix system.
