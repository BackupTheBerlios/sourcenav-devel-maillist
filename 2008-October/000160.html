Hi,

if you've built as shown in INSTALL (./configure
--prefix=/opt/sourcenav) then just remove that directory. You might
also remove your ~/.sn directory where settings are saved.

If you've not built with --prefix then data truly is shattered within
your /usr filesystem and there is no comfortable way to deinstall it,
I'm sorry.
That is why FSSTND compliance is planed for the next release NG4 and
we might also release a .rpm to make installation and removal easy.
