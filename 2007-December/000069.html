Hi all,

as you are currently touching the DB code and preparing for NG3,
can you ensure that the new DB code will support >2 GB databases?

I remember that I had problems with even NG2 (using Linux 32bit
or 64bit) when cross referencing a really big project (>10M LOC).
During the xref phase, a dialog with the follow error message did
show up:

"dbopen error:/home/xca1019/temp/SOURCES/.snprj/FOO.by:Invalid argument"

Checking the file size showed 2147475456 bytes, which is 8K short
of 2 GB. So I rebuild the tool with explicit 64bit file handling:

export CFLAGS="-D_LARGEFILE_SOURCE -D_LARGEFILE64_SOURCE -D_FILE_OFFSET_BITS=64"
./configure --prefix=/opt/sourcenavigator-NG2

This did the trick for me. Ideally one should add those global
defines to the appropriate autoconf files. I haven't done a patch
on that yet, but if you anyway replace the DB code you might
want to add that.

Thanks
Daniel
